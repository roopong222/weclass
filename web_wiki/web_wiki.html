1.html

1.1기본프레임

코딩 - 코드를 입력해서 소프트웨어 제어하는 행위
html - 홈페이지에 들어가는 컨텐츠를 입력하는 언어
css - 홈페이지의 스타일을 담당하는 언어
javascript - 홈페이이의 기능을 담당하는 언어
jquery - javascript 언어를 보다 쉽게 사용하기위해 개선된 언어

통신흐름

브라우저 - 크롬,익스플로러,사파리,파이어폭스...등등
클라이언트 - 요청자 요청을 하는 주체 -> 브라우저가 클라이언트에 속한다
서버- 제공자(저장소) 파일들이 저장되는 공간. 클라이언트의 요청에 응답하여 파일을 제공해주는 역할

기본 특징

1.html파일 안에엇 엔터를쳐서 줄바꿈을 하더라도 줄바꿈되지 않는다

2.html파일안에서 띄어쓰기를 여러번 입력하더라도 한번으로 취급된다.

3.브라우저 클라이언트는 다운받은 html파일을 위에서 아래 왼쪽에서 오른쪽으로 해석해 나가며 해석한 결과를 브라우저에 출력한다

4.대부분의 태그는 시작과 마침태그로 구성되어 있다.

5.모든html파일은 눈에보이는 컨텐츠를 다루는 공간 body태그와 눈에 보이지않는 컨텐츠를 다루는 공간 head태그로 구성되어 있다

6.html파일은 기본프레임을 작성해야 한다. 그래야 의도한 설정을 해줄 수 있다.(한글이 깨지지않게 하는방법, 탭제목을 수정하는 방법)

7.html에서는 포함관계가 중요하다. -> 족보관계가 형성이 된다.

8.족보관계를 명확하게 표현하기위해 tab키를 들여쓰기를 해준다.(shuft+tab키를누르면 다시 밖으로 당길 수 있다.)

1.2최소문맥태그

문맥태그 : h1 ~ h6,p,address,blockquote 등등

문맥태그는 사용할떄마다 자동줄바꿈이 된다

(p) 장문,단문,문단,설명 컨텐츠 (/p)
(address) 주소컨텐츠 (/address)
(blockquote) 인용글컨텐츠 (/blockquote)

강조태그 : strong,b,em,i,span,sub,sup 등등

주석 (ctrl+/) !-- 메모가능 / 브라우저가 해석을안한다

1.3링크태그

홈페이지를 사용하는 사용자가 클릭했을 때 목적지페이지로 이동시켜주는 태그

a태그는 href속성이 필수적으로 사용되어야 하며 다른 속성들은 상황에 맞춰 선택적으로 사용할 수 있다

a태그는 강조태그처럼 클릭했을 때 링크시켜야되는 단어나 문장에 자유롭게 사용가능하다.

네이버로 바로가기

절대경로

절대경로란 변하지 않는 경로

웹 상에서 긁어온 주소를 절대경로라고 한다

상대경로

상황에 따라 표현 값이 변할 수 있는 경로

현재파일에서 목적지파일까지의 경로

. ->현재 파일이 위치한 폴더의 위치를 불러옴
.. ->현재 파일이위치한 폴더의 상위폴더위치를 불러옴
/ ->폴더를 열고 들어감

속성(attribute)

속성은 언제나 시작태그에 한칸띄어서 구분하여 입력한다.

속성은 태그에 2개이상 여러개가 사용될 수 있다. 입력하는 순서는 상관없다.

속성표현법 - [속성 명 = "속성 값"]

=등호는 등호 기준 우측의 값을 왼쪽 속성에 적용(대입)해주는 대입연산자로 사용된다.

속성은 글로벌과 로컬속성으로 나뉜다 href속성은 a태그에만 사용할수 있는 로컬속성이며 id라는 속성은 모든태그에 적용할수 있는 글로벌 속성이다

a태그의 여러가지 사용법

선생님 전화번호
파일 다운로드
책갈피기능 사용 속성타이틀로 이동

1.4이미지,영상

이미지,비디오

이미지태그

src속성에 이미지파일까지의 경로를 입력하여 작성한다

alt속성에는 이미지를 설명할수있는 대체텍스트를 제공해야 한다

연결할 수 있는 이미지파일의 형태는 일반적인 이미지라면 대부분 가능하다(jpg, png, gif, svg)

이미지태그는 일반적인 텍스트 글자와 같은 컨텐츠 취급된다. 우리가 글자를 읽고 해석한뒤 제목이라는 문맥 혹은 설명이라는 문맥으로 결정하듯이 이미지도 전체적인 맥락안에서 제목 또는 설명컨텐츠로 구분될 수 있다

비디오태그

비디오파일은 브라우저와 아직 호환성이 완벽하지 않기 때문에 mp4파일은 거의 모든 브라우저에서 호환이 가능하여 브라우저에서 볼 수 있지만 다른형태의 파일은 브라우저에서 호환되지 않는경우가 많다

비디오태그의 마침태그가 있는 이유는 자식태그로 source태그를 구성하여 여러가지 형태의 비디오 타입을 수용할 수 있게 하기 위해서이다.

비디오 태그는 자식태그 source태그를 위에서 아래로 탐색해나가면서 해당하는 브라우저와 호환되는 영상이라면 바로 플레이 한다.

비디오 태그의 속성 (controls, autoplay, muted, loop, poster="#")

1.5리스트

홈페이지에서 구조와 형태가 비슷한 연관되어있는 컨텐츠를 표기하는 방법

li태그는 단독적으로 사용할 수 없다. 반드시 순서의 흐름을 고려하여 순서가 있다면 부모태그에 ol태그를 순서가 없다면 부모태그에 ul태그를 사용해야 한다.

ol/ul태그의 자식태그로는 li태그만 사용할 수 있다.

li태그 안에는 또 다른 문맥들이 형성될 수 있다. 그동안 배운 제목,설명태그 포함해서 여러가지 태그문맥이 li태그안에서 사용될 수 있다.

ordered list - ol

연관된 컨텐츠 즉 리스트로 표현되는 컨텐츠사이에 순서의 영향이 있는 경우 ol태그를 부모태그로 사용한다

unordered list - ul

연관되어있는 컨텐츠 즉 li태그들이 서로 순서의 영향이 없을 때 부모태그로 ul태그를 사용한다

리스트의 중첩

홈페이지에서 메뉴단을 만들 때 리스트를 중첩하여 depth구조를 만들 수 있다

tip]1뎁스를 다 완성한 뒤 2뎁스를 만들고 2뎁스를 다 완성한뒤 3뎁스를 만들 것

1.6정의형리스트

정의할 대상에 대한 설명을 표현하는 태그

정의할 대상을 dt태그를 통해 표현하고 dt태그 다음에 나오는 dd태그는 이를 설명하는 맥락을 가지고 있다

dt태그 다음에 오는 dd태그는 여러개가 될 수 있다.

dt태그도 여러개가 사용될 수 있다.

반드시 dt태그가 만들어진 후 dd태그가 만들어져야 하는 구조이다.

dd태그안에는 또다른 문맥이 만들어질 수 있다

주로 쇼핑몰에서는 상품에 대한 정보를 dl태그로 표현한다.(예:가격정보,재질정보...)

주로 레시피사이트에서는 음식에대한 정보를 dl태그로 표현한다(당근:~~g)

용어사전사이트에서도 용어에 대한 설명을 dl태그로 표현한다

1.7영역구성-div

영역을 나눌 때 사용하는 태그

컨텐츠의 맥락에 따라서 영역을 나눔

영역구성태그는 자유롭게 사용할 수 있으나, 좌우로 뚜렷하게 나뉘어진 컨텐츠는 무조건 영역구성태그를 사용하여 나눈다.

영역의 이름은 자유롭게 사용가능하다. (주로사용하는 이름 : container, header, aside, contents)

css에서 나누어진 영역단위로 배치를 할 수 있기 때문에 좌우로 나뉘어진 영역은 반드시 영역태그로 구성되어야 한다.

div로 부터 파생되어 나온 영역구성 태그들

영역의 쓰임새에 따라 태그명을 새롭게 만듬

  • header - 홈페이지의 머릿글 컨텐츠에 사용되는 영역태그(로고,메뉴)
  • footer - 홈페이지의 하단글 컨텐츠에 사용되는 영역태그(회사정보)
  • main - 홈페이지의 메인컨텐츠를 전반적으로 다룰 때 사용(main태그는 html파일당 1개만 사용 가능)
  • section/article - 구획을 나눌 때 사용
  • nav - 홈페이지의 메뉴단을 만들 때 사용하는 영역태그
  • figure - 이미지나 영상의 전용 영역태그
  • aside - 홈페이지의 독립된 사이드컨텐츠를 만들 때 사용하는 영역태그 (전체컨텐츠의 공간(영역)안에 넣지않아도 된다.)

1.8표

table태그를 구성하면 보이지않는 표공간이 생성된다.

table태그안에서 컨텐츠를 입력하는 방법은 가로 행(row)을 먼저 만들고 행안에 왼쪽부터 셀들을 입력하면 된다.

이 때, 입력하는 셀들이 일반 내용셀이라면 td태그를 사용하고 제목셀이라면 th태그를 사용한다.

셀 병합

병합이 시작되는 셀을 찾아서 가로병합(colspan)/세로병합(rowspan)속성을 사용한다. 이 때, 속성에 적용되는 값은 병합하느 셀의 개수를 숫자로 입력한다. 그리고 병합이 되어져서 사라지는 셀은 태그를 지워준다.

tip : 병합되기 전 모습의 표를 먼저 완성한 후 최종적으로 병합을 한다.

1.9폼

홈페이지를 사용하는 사용자가 내용을 입력할 수 있도록 미리 제공하는 양식태그

label 태그는 이름표태그로 사용하며 사용자가 입력하는 입력박스에대한 이름을 달아줄때 사용한다.

label태그와 input태그를 서로 연결하기 위해서는 input태그의 id값을 label태그의 for속성에 그대로 적용한다. - 홈페이지의 사용 편의를 위해 자주 사용하는 패턴

label태그가 디자인적으로 보이지 않더라도 실제로 html태그 label태그는 구성해야하며 css에서 안보이게 처리하는 방법을 사용한다.

input type속성이 radio인 경우 select태그와 같이 제한된 내용을 입력하게 끔 유도할 때 사용하며, radio버튼의 name속성 값을 동일하게 하면 여러개의 radio버튼중 1개만 선택되게 할 수 있다

input type="text" 태그의 경우는 짧은 한줄의 텍스트를 입력시킬 때 사용하며, textarea태그는 장문의 텍스트를 입력시킬때 사용한다.

input type

  • text
  • password
  • radio
  • checkbox
  • submit
  • reset

사용되는 속성들

  • value - 입력박스에 들어가는 실제 정보
  • disabled - 선택한 입력박스를 비활성
  • required - 선택한 입력박스의 실제정보를 반드시 입력시킬 때 사용
  • selected - 셀렉트박스에서 선택한 option태그가 선택된상태로 보이게 할 때 사용
  • checked - 체크박스/라디오 버튼이 체크된 상태로 표현할 때

2. css

2.1css기초

css는 html태그를 스타일링하는 style언어

css 입력방식

  • inline-style(element-style)

    태그를 직접 선택하여 시작태그에 style이라는 속성을 해서 스타일을 입력하는 방법

  • 내부스타일시트

    head태그안에 style태그를 작성하여 스타일을 입력하는 방법

  • 외부스타일시트

    html파일 바깥에 별도의 css파일을 생성해서 스타일을 입력하는 방법

css 기초문법

css는 태그를 선택해서 스타일을 입력하는언어 그렇기 때문에 선택하는 부분과 스타일이 입력되는 부분으로 나뉜다

선택자{스타일속성:스타일속성값 스타일속성:스타일속성값;}

;(세미콜론)은 구분자로 사용된다.

선택자(태그명,클래스명,아이디명으로 선택할 수 있다.)

단계적으로 적용되는 css스타일

css스타일을 해석하는 주체는 브라우저이며 마찬가지로 위에서 아래 왼쪽에서 오른쪽으로 해석한다.

새로운 스타일을 적용할 때 이전 스타일속성은 새로운 속성값은 입력하지 않았다면, 이 전 스타일속성값이 그대로 적용된다.

아이디,클래스이름(식별자코드)을 사용한 선택자 구성

태그에 부여한 이름을 통해 선택자를 구성할 수 있다.

태그명을 사용하여 선택자부분에 표현할 수 있다.

클래스는 접근자(.)을 사용하여 선택자부분에 표현할 수 있다.

아이디는 접근자(#)을 사용하여 선택자부분에 표현할 수 있다.

선택자 부분에서 선택된 태그는 기본적으로 복수형태이다.

태그의 클래스명은 2개이상 여러개를 사용할 수 있으며, 띄어쓰기를 사용하여 클래스를 추가한다.

관계선택자

선택자부분에 태그의 족보관계를 사용해서 구성할 수 있다.

자식선택자

선택자부분에서 ">"를 상용하면 (~안에 포함되어있는) 자식중에서 해석한다

자손선택자(하위선택자)

선택자부분에서 띄어쓰기를 사용하면 (~안에 포함되어있는) 자손중에서 해석한다

다음형제선택자

선택한 태그를 기준으로 다음형제를 선택할 수 있다.

선택자부분에서 +,~을 사용하여 다음형제를 선택할 수 있다.

선택자부분에서 +를사용하면 앞에 선택된 태그의 다음형제 하나만 선택한다.

선택자부분에서 ~을 사용하면 앞에 선택된 태그의 다음형제 여러개를 선택한다.

속성선택자

태그가 갖고있는 속성을 사용하여 선택자를 구성할 수 있다

선택자 부분에[ ]대괄호안에 속성명,속성값을 입력하여 선택할수있다

순번선택자

태그가 갖고있는 순서를 이용하여 선택할 수 있다.

태그명:nth-of-type() 괄호안에는 수식/절대값을 입력하여 선택할 수 있다.

태그명:last-of-type()/first-of-type 을 통해 마지막과 첫번째를 선택할 수 있다.

선택자의 우선순위

태그를 선택하는 선택자부분은 우선순위 점수라는 개념이 있으며, 이 우선순위점수가 높을수록 최종스타일로 반영된다.

  • 태그명으로 선택 - 우선순위 1점
  • 클래스명으로 선택 - 우선순위 10점
  • 아이디명으로 선택 - 우선순위 100점
  • 인라인스타일방식 - 우선순위 1000점
  • css에서 !important를 사용하면 무조건 스타일을 적용하게 된다(단 남용해선 안된다.)

선택자의 우선순위 점수는 합산된다.

부모태그가 자식태그에게 스타일을 물려주는 행위

2.3display - 태그의 형태

태그의 생김새 - 기본적으로 직사각형

전제 - 우리가 입력한 태그는 먼저 만들어진 순서대로 위에 하나씩 쌓이는 거꾸로하는 테트리스 같다

  • block - 90%
  • inline - 8%
  • inline-block - 2%

block - 블록레벨태그

display:block; 으로 설정되어있음.

포함되는 컨텐츠의 양과 관계없이 무조건 부모태그 만큼의 너비를 다 차지하는 형태

br태그를 사용하지 않아도 부모태그의 너비를 다 차지하고 있기 때문에 자동으로 줄바꿈이 된다.

박스형태의 태그 - 가로와 세로사이즈를 지정할 수 있다.

h1~h6, p, address, blockquote, div, ul, ol, li, dl, dt, dd, header, nav, footer, section, article, aside,...

inline - 인라인레벨태그

display:inline; 으로 설정되어있음

포함되는 컨텐츠의 양만큼 공간을 차지하는 형태

포함하는 컨텐츠의 양만큼 공간을 차지하기 때문에 줄바꿈되지않고 나란히 배치될 수 있다

라인형태이기 때문에 가로와 세로사이즈를 지정해도 변화가 없다.

span,i,em,strong,b,sup,sub,a,input,label,...

inline-block - 인라인블록레벨 태그

display:inline-block; 으로 설정되어있음

블록레벨과 인라빈레벨특징을 반반 섞은 형태

인라인레벨태그처럼 포함하는 컨텐츠의 양만큼 공간을 차지하여 나란히 배치가 될수 있다.

블록레벨 태그처럼 가로와 세로사이즈를 지정할 수 있는 박스형태다

img, video, input, select, textarea

박스의 크기

모든 태그는 컨텐츠가 입력되는 공간인 content-box가 있다.

우리가 width/height속성을 사용하여 크기를 지엉하면 이 컨텐츠가 입력되는 공간인 content-box의 크기를 변경하는 것이다

이 컨텐츠 박스는 단순한 글자나 이미지 뿐만이 아니라 자식태그가 들어오는 공간이기도하다. 자식태그가 포함된다면 자동으로 부모태그만큼의 너비가 상속된다.

box-sizing

기본적으로 box-sizing속성은 content-box값으로 설정되어 있다. 이 때 너비/높이를 지정하면 우리가 지정한 너비와 높이는 테두리 안쪽의 컨텐츠가 입력되는 공간의 너비와 높이를 설정하게 된다. 하지만 box-sizing속성을 border-box로 설정하고 너비와 높이를 수정하면 테두리가 굵기까지 포함해서 너비와 높이를 설정하게된다.

태그작성 포함관계 규칙

  • 인라인레벨 태그는 자유롭게 서로 포함하거나 포함될 수 있다.
  • 인라인레벨 태그는 블록레벨태그를 포함할 수 없다(단 a태그는 포함 가능하다)
  • 블록레벨태그는 서로 자유롭게 포함하거나 포함될 수 있지만(문맥에 맞는경우만)
  • 블록레벨태그는 인라인레벨 태그를 포함할 수 있다

2.4boxmodel

각각의 태그는 테두리와, 테두리 기준 바깥여백, 태두리 기준 안쪽여백을 갖고 있다.

각각의 태그는 브라우저마다 조금씩 박스모델값의 차이가 있다. 그래서 스타일을 조정할 때는 이미 설정되어있는 박스모델을 초기화 해주는 작업이 필수적이다.

  • border - 태그의 경계선
  • padding - 테두리 안쪽 여백
  • margin - 테두리 바깥 여백

border - 테두리

태그의 경계선이며, html문서는 먼저 만들어진 태그가 윗쪽부터 하나씩 적재 되는 구조이다. 이때, 적재되는 기준선으로 사용된다.

padding - 테두리 안쪽 여백

테두리 기준 안쪽 여백을 설정하면 컨텐츠가 입력되는 공간인 content-box를 밀어낸다.

margin - 테두리 바깥 여백

테두리 기준 바깥 여백을 설정하면 인접한 태그를 밀어내거나, 혹은 밀지못하는 상황이라면 선택된 태그가 밀릴 수 있다.

doxmodel- 축약형 표현

margin/padding 축약형 표현 규칙은 같다.

  • 2개입력(상,하)(좌,우) - margin: 50px 100px ;
  • 3개입력(상)(좌우)(하) - margin: 50px 200px 100px;
  • 4개입력(상)(우)(하)(좌)[시계방향] - margin: 50px 100px 150px 200px;

margin의 상쇄 규칙/padding(상쇄없음)

margin은 테두리 기준 바깥쪽 여백이므로 인접한 태그를 밀어내거나 밀릴 수 있다

바깥쪽 여백이기 떄문에 인접한 태그 사이에는 상쇄 규칙이 적용된다.(인접한 두개의 태그의 margin값 중 근 값이 적용된다)

상하 margin만 상쇄된다.(좌우 margin은 상쇄되지 않는다.)

inline레벨 태그에서의 margin과 apdding

inline레벨 태그는 박스형태가 아니며, 라인형태이다. 그래서 마진과 패딩이 독특하다.

inline레벨태그에서는 좌,우마진은 제한적으로 적용되고 상하값은 적용되지 않는다.

imline레벨태그에서는 좌,우패딩은 제한적으로 적용되고,상하값은 지정하지 않는다.

2.5width_height

width

width라는 속성으로 선택한 태그의 가로사이즈를 지정할 수 있다.

box-sizing:content-box (기본값)상태에서 width를 지정하면 컨텐트가 놓여지는 공간의 가로사이즈가 적용된다

width의 기본값은 auto이다.

auto->상황에따라 자동으로 변동될 수 있는 상태

너비에 사용되는 단위는 px, %, vw, vh 사용될 수 있다

  • %

    %단위는 상대단위 이다. 부모태그의 너비의 비율단위입니다. 현재 box1클래스의 부모태그는 body태그이고 body태그는 브아우저의 크기를 조절하면 자동으로 같이 줄거나 늘어나는 상태이기 때문에, 그 상태는 50%비율로 너비가 계산되므로 브라우저에 크기에 절반만큼의 비율을 가진 너비로 설정 된다.

    현재 부모태그인 parent3클래스영역은 80%의 너비가 설정되어있다. 이는 부모인 body태그 너비의 80%만큼을 계산한다라는 뜻이고, body태그는 사용자에 의해서 줄어들거나 늘어날 수 있는 유동적인 너비를 갖고있기 때문에 자식 p태그도 부모인 parent3클래스의 50%의 너비를 갖고있어서 덩달아서 줄어들거나 늘어날 수 있는 상태가 된다.

  • vw

    이 태그는 width:30vw로 설정되어있다. vw단위는 view-port-width라는 뜻으로 view-port는 브라우저의 화면을 뜻한다. 즉 브라우저 화면의 너비에대한 비율 단위이다. 그렇기 때문에 이 태그는 브라우저를 줄이고 늘일때마다 변경된다.

  • vh

    이 태그는 width:50vh로 설정되어있다. vh단위는 view-port-height라는 뜻으로 브라우저 화면의 높이에대한 비율 단위이다. 그렇기 때문에 이 태그의 너비는 브라우저의 세로사이즈 를 줄이고 늘일 때 마다 변경된다.

height

height라는 속성으로 태그의 세로사이즈를 설정할 수 있다. 단위는 지금까지 배운 모든 단위를 사용할 수있다.(px,en,rem,vw,vh,%)

height의 기본값은 auto이며

box-sizing:content-box (기본값)으로 설정이 되어있을 때 height가 의미하는 것은 컨텐츠가 들어가는 공간인 컨텐트 박스의 높이이다.

이 태그는 가로사이즈와 세로사이즈를 200픽셀로 설정했다. 그렇기 떄문에 컨텐츠가 들어가는 컨텐츠박스의 크기가 200X200픽셀로 설정된 것이다. 그러나 안에 들어가는 내용이 해당하는 컨텐츠박스의 크기를 넘어설만큼의 긴 내용이 들어간다면 컨텐츠 박스 바깥으로 컨텐트가 흘러넘칠 수 있다. 이 흘러넘치는 컨텐츠를 기존의 거꾸로하는 테트리스 구조에서 처럼 적재되는 공간을 차지하지 않는다.

너비만 200픽셀로 지정했고 높이에 대한 언급은 없었기 때문에 기본값 auto가 적용된다. auto값은 포함되는 컨텐츠의 양에 따라 자동으로 높이를 조절한다는 뜻이므로 box2영역의 높이를 제한했을 때 처럼 컨텐츠가 흘러넘치는 경우가 없다.

2.6overflow

overflow - 흘러넘친 컨텐츠에 대해서 대응하기위한 css속성이다.

  • hidden - 흘러넘친 컨텐츠에 대해서 안보이게 처리
  • scroll - 흘러넘친 컨텐츠에 대해서 스크롤바를 표현
  • auto - 흘러넘친 컨텐츠에 대해서 스크롤바를 표현
  • visible(기본값) - 흘러넘친 컨텐츠를 그대로 보여주도록 처리

2.7background

태그의 공간에 배경색 또는 배경이미지를 나타낼 때 사용하는 css속성

  • background-color(transparent) -> 배경색상을 입력할 때 사용하는 속성
  • background-image(none) -> 배경이미지를 입력할 때 사용하는 속성
  • background-repeat(repeat) -> 입력된 배경이미지의 가로세로반복속성을 제어하는 속성
  • background-position(0px 0px)-> 입력된 배경이미지의 위치를 x/y축값으로 제어하는 속성

background-position

background-position속성은 기존 좌상단(0,0)좌표로 생각 했을 때 좌표체계에 따라서 픽셀단위로 수치를 입력하여 배경이미지를 변경시키는 방법이 있다. 하지만 그런 방법만 통해서 배경이미지를 선택한 영역의 가로기준 중앙 세로기준 중앙에 배치하려고 하면 계산하는 과정이 복잡하기 때문에 키워드 방식으로도 배경이미지를 배치할 수있다.(left, right, top, bottom)

키워드 입력 방식으로만 배경이미지의 위치를 변경하기에는 표현에 제약이 많이 따르기 때문에 %단위를 사용해서도 배경이미지를 배치할 수 있다. 이 때, 좀 더 세부적인 위치변경을 하기위해서 calc()라는 css상의 계산기를 활용할 수 있다.

calc() 괄호안에는 +(더하기), -(빼기), *(곱하기), /(나누기)기호를 통해 계산을 할 수 있다. 이 때 더하기와 빼기는 부호 앞뒤로 반드시 띄어쓰기를해서 표현해야 하며, 곱셈 나눗셈은 띄어쓰기를 하지않아도 된다.

background-size

background-size:(가로사이즈) (세로사이즈) 속성을 통해서 배경이미지의 가로와 세로사이즈를 지정할 수 있다.

이 때, 가로사이즈를 픽셀이 아닌 %를 사용할 수 있다.

그리고, auto라는 키워드를 통해서 원본이미지가 갖고있는 비율계산을 자동으로 적용해줄 수 있다.

  • cover -> 원본이미지의 가로세로비율을 유지하면서 선택한 태그의 공간을 빈틈없이 가득 메꾸는 방식(이 때 배경이미지가 잘려나가도 신경쓰지 않는 방식)
  • contain -> 원본이미지의 가로세로비율을 유지하면서 선택한 태그의 공간을 최대한 가득 메꾸는 방식(이 때 배경이미지가 절대 잘려나가지 않는 선에서 사이즈를 조절하는 방식)
